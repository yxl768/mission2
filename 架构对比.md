# Seq2Seq 深度学习架构对比

## 概述

本项目实现了两种序列到序列（Seq2Seq）深度学习架构，用于将分子的IUPAC名称翻译为SELFIES表示。两种架构各有特点，本文从架构设计、核心组件、训练策略、性能表现等维度进行全面对比。

***

| ---------------------- | -------------- | -------------- | ------------------------- | --------------- | ---------------------------------------------------------------- |
| **Seq2SeqGRU**         | 双向GRU          | 单向GRU          | Bahdanau（加性注意力）           | 拼接双向GRU最后时刻隐藏状态 | [models.py](file:///d:/mission2/seq2seq_mol/models.py#L71-L207)  |

## 2、核心组件对比

### 2.1 编码器对比

#### GRUEncoder

```python
class GRUEncoder(nn.Module):
    def __init__(self, vocab_size, embed_size, hidden_size, num_layers=1, 
                 dropout=0.1, bidirectional=True):
        self.embedding = nn.Embedding(vocab_size, embed_size)
        self.gru = nn.GRU(
            embed_size, hidden_size,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0.0,
            bidirectional=bidirectional,
        )
```

**核心特点**：

- 使用双向GRU，同时从左到右和从右到左处理序列
- 使用PackedSequence处理变长序列，跳过padding部分
- 输出包含序列每个位置的隐藏状态和最后时刻的隐藏状态

#### Transformer编码器

```python
self.transformer = nn.Transformer(
    d_model=embed_size, nhead=num_heads,
    num_encoder_layers=num_encoder_layers,
    num_decoder_layers=num_decoder_layers,
    dim_feedforward=dim_feedforward,
    dropout=dropout,
    batch_first=True,
)
```

**核心特点**：

- 由多个编码器层堆叠而成，每层包含多头自注意力和前馈网络
- 使用位置编码显式建模序列顺序
- 全并行计算，不依赖循环结构

### 2.2 注意力机制对比

#### Bahdanau Attention（加性注意力）

```python
class BahdanauAttention(nn.Module):
    def __init__(self, hidden_size, enc_hidden_size):
        self.W_a = nn.Linear(enc_hidden_size, hidden_size, bias=False)
        self.U_a = nn.Linear(hidden_size, hidden_size, bias=False)
        self.v_a = nn.Linear(hidden_size, 1, bias=False)
    
    def forward(self, decoder_hidden, encoder_outputs, padding_mask=None):
        scores = self.v_a(torch.tanh(self.W_a(encoder_outputs) + 
                                      self.U_a(decoder_hidden.unsqueeze(1))))
        weights = torch.softmax(scores, dim=1)
        context = (encoder_outputs * weights).sum(dim=1)
        return context, weights
```

**计算公式**：

```
score_i = v_a^T · tanh(W_a · h_i + U_a · s_t)
α_i = softmax(score_i)
c_t = Σ α_i · h_i
```

**特点**：

- 单头注意力，计算一个注意力权重分布
- 加性计算，适合编码器和解码器隐藏维度不同的情况
- 每个解码步骤重新计算注意力

#### Multi-Head Self-Attention（多头缩放点积注意力）

**计算公式**：

```
Q = X · W_Q, K = X · W_K, V = X · W_V
head_i = Attention(Q_i, K_i, V_i) = softmax(Q_i·K_i^T / √d_k) · V_i
MultiHead = Concat(head_1, ..., head_h) · W_O
```

**特点**：

- 多头并行计算，捕捉不同类型的依赖关系
- 缩放点积注意力，计算效率更高
- 自注意力机制，同时建模序列内部所有位置关系

### 2.3 解码器对比

#### GRUDecoder

```python
class GRUDecoder(nn.Module):
    def __init__(self, vocab_size, embed_size, hidden_size, num_layers=1, dropout=0.1):
        self.embedding = nn.Embedding(vocab_size, embed_size)
        self.gru = nn.GRU(embed_size + hidden_size, hidden_size, batch_first=True)
        self.output_projection = nn.Linear(hidden_size * 2, vocab_size)
    
    def forward(self, input_ids, hidden, context):
        embedded = self.embedding(input_ids)
        gru_input = torch.cat([embedded, context.unsqueeze(1)], dim=-1)
        output, hidden = self.gru(gru_input, hidden)
        combined = torch.cat([output, context.unsqueeze(1)], dim=-1)
        return self.output_projection(combined), hidden
```

**特点**：

- 输入拼接：当前token嵌入 + 注意力上下文向量
- 输出拼接：GRU输出 + 上下文向量
- 循环结构，逐token生成

#### Transformer解码器

```python
output = self.transformer.decoder(
    self._embed_target(tgt_input),
    memory,
    tgt_mask=causal_mask,           # 因果掩码，防止看到未来token
    tgt_key_padding_mask=target_padding,
    memory_key_padding_mask=source_padding,
)
```

**特点**：

- 掩码多头自注意力：确保生成第i个token时只能看到前i-1个token
- 编码器-解码器注意力：关注编码器输出的相关部分
- 全并行计算，一次性处理整个目标序列

### 2.4 位置编码对比

#### GRU（隐式位置编码）

GRU通过循环结构天然地编码位置信息：

- 序列顺序由循环处理顺序决定
- 位置信息隐式地包含在循环状态转移中

#### Transformer（显式位置编码）

```python
class PositionalEncoding(nn.Module):
    def __init__(self, embed_size, max_length=2048):
        position = torch.arange(max_length, dtype=torch.float32).unsqueeze(1)
        frequencies = torch.exp(
            torch.arange(0, embed_size, 2, dtype=torch.float32) * 
            (-math.log(10000.0) / embed_size)
        )
        encoding = torch.zeros(max_length, embed_size)
        encoding[:, 0::2] = torch.sin(position * frequencies)
        encoding[:, 1::2] = torch.cos(position * frequencies)
        self.register_buffer("encoding", encoding.unsqueeze(0))
    
    def forward(self, values):
        return values + self.encoding[:, :values.size(1)]
```

**特点**：

- 使用正弦/余弦函数为每个位置分配唯一标识
- 偶数位置用正弦，奇数位置用余弦
- 位置编码与词嵌入相加，不增加额外参数

## 3、分子表示提取对比

### 3.1 Seq2SeqGRU 的表示提取

```python
def encode(self, src_ids, src_lengths):
    encoder_outputs, encoder_hidden = self.encoder(src_ids, src_lengths)
    # 取双向GRU的最后两层隐藏状态
    final_hidden = encoder_hidden[-2:]  # (2, batch_size, hidden_size)
    combined = torch.cat([final_hidden[0], final_hidden[1]], dim=-1)
    return combined  # (batch_size, 2*hidden_size)
```

**策略**：

- 提取双向GRU最后时刻的隐藏状态
- 拼接前向和后向隐藏状态
- 维度：`2 * hidden_size`（默认256维）

**优缺点**：

| 优点           | 缺点        |
| ------------ | --------- |
| 只使用最终状态，计算简单 | 丢失中间位置信息  |
| 适合短序列建模      | 长序列信息衰减严重 |

### 3.2 TransformerSeq2Seq 的表示提取

```python
def encode(self, src_ids, src_lengths=None):
    memory, padding = self._memory(src_ids)
    valid = (~padding).unsqueeze(-1)
    return (memory * valid).sum(dim=1) / valid.sum(dim=1).clamp_min(1)
```

**策略**：

- 对编码器输出进行均值池化
- 只考虑非padding位置
- 维度：`embed_size`（默认256维）

**优缺点**：

| 优点       | 缺点           |
| -------- | ------------ |
| 利用所有位置信息 | 可能对长序列中的噪声敏感 |
| 表示更充分    | 计算量较大        |

## 7、架构优缺点总结

### 7.1 Seq2SeqGRU

**优点**：

1. **参数效率高**：模型参数较少，适合小数据集
2. **训练稳定**：循环结构收敛速度快，梯度消失风险相对较小
3. **实现简单**：架构直观，易于理解和调试
4. **内存效率高**：不需要存储整个序列的中间结果

**缺点**：

1. **长距离依赖建模能力弱**：循环结构存在梯度消失问题，难以捕捉长距离依赖
2. **并行计算能力差**：循环结构无法并行化，训练速度慢
3. **表示能力有限**：仅使用最后时刻的隐藏状态，丢失中间位置信息
4. **对冲突监督信号敏感**：在增强数据训练时容易出现表示崩溃

### 7.2 TransformerSeq2Seq

**优点**：

1. **长距离依赖建模能力强**：注意力机制可以直接建模任意两个位置之间的关系
2. **并行计算能力强**：全并行结构，训练效率高
3. **表示能力强**：均值池化利用所有位置信息，表达更充分
4. **泛化能力好**：在增强数据上表现稳定，不易出现表示崩溃
5. **现代NLP主流架构**：扩展性好，容易迁移到其他任务

**缺点**：

1. **参数数量大**：多头注意力和前馈网络增加了模型参数
2. **训练不稳定**：需要精心调参，对学习率和batch size敏感
3. **内存消耗大**：需要存储整个序列的中间结果
4. **位置信息依赖显式编码**：需要额外的位置编码机制

## 8、架构选择建议

### 8.1 选择Seq2SeqGRU的场景

| 场景     | 理由          |
| ------ | ----------- |
| 小数据集   | 参数效率高，训练稳定  |
| 短序列任务  | 循环结构适合短序列建模 |
| 快速原型验证 | 实现简单，调试方便   |
| 低资源环境  | 内存效率高，训练时间短 |

### 8.2 选择TransformerSeq2Seq的场景

| 场景      | 理由           |
| ------- | ------------ |
| 长序列任务   | 注意力机制捕捉长距离依赖 |
| 大数据集    | 并行计算能力提升训练效率 |
| 高质量生成任务 | 多头注意力提升表示质量  |
| 生产环境部署  | 泛化能力强，性能稳定   |

### 8.3 建议

基于本项目206条训练数据的实验结果：

1. **推荐使用Transformer模型**：在翻译任务和性质预测任务上均显著优于GRU
2. **数据增强有效**：增强数据提升了模型的泛化能力，Transformer在增强数据上表现稳定
3. **注意数据泄漏**：增强数据需按IUPAC名称去重，否则评估结果虚假
4. **GRU需配合PCA降维**：原始GRU嵌入在性质预测任务上表现较差，PCA降维后性能显著提升
